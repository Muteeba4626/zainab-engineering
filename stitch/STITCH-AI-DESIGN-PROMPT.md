# 🛠️ STITCH AI BEST PRACTICES & TIPS

## Getting Started with Stitch AI

### 1. Start with Structure, Not Style
**Do This First:**
- Begin by describing the page structure and layout
- Specify the number of sections
- Define the content hierarchy
- THEN add style details

**Example Prompt Structure:**
```
"Design a hero section with:
- Split layout (60/40)
- Left: headline, subtext, 2 buttons
- Right: 3D product visualization
- Background: gradient mesh"
```

### 2. Be Specific with Colors
**Don't Say:** "Use blue colors"
**Do Say:** "Use Electric Blue #0066FF for primary buttons and Deep Navy #0A1628 for headers"

### 3. Reference Real-World Examples
- Mention specific websites for style reference
- Name brands with similar aesthetics
- Example: "Similar energy to Tesla's website but for industrial equipment"

### 4. Iterate in Sections
- Don't try to generate the entire website at once
- Design one section/page at a time
- Refine each section before moving on

### 5. Use Descriptive Adjectives
**Weak:** "Make it look modern"
**Strong:** "Futuristic with sleek lines, subtle 3D depth, and floating elements with soft shadows"

### 6. Specify Negative Space
- Mention desired whitespace
- "Generous padding between sections"
- "Breathing room around text blocks"

### 7. Animation Instructions
- Describe the motion, not just "animate it"
- "Cards lift 8px on hover with a 0.3s ease-out transition"
- "Text reveals letter-by-letter from left"

### 8. Mobile Considerations
- Always mention mobile layout changes
- "On mobile, stack these columns vertically"
- "Hide 3D elements on devices under 768px"

---

## Common Mistakes to Avoid

### ❌ Mistake 1: Vague Instructions
**Bad:** "Make the products section look nice"
**Good:** "Create a 3-column product grid with cards that have rounded corners, subtle shadows, product image at top, title below, and a hover state that lifts the card"

### ❌ Mistake 2: Conflicting Requirements
**Bad:** "Make it minimalist but with lots of features and animations"
**Good:** "Clean minimalist layout with strategic animation on key interactions only"

### ❌ Mistake 3: Ignoring Hierarchy
**Bad:** "Put all this content on the page"
**Good:** "Hero headline is most prominent, followed by supporting text, then feature cards"

### ❌ Mistake 4: No Context
**Bad:** "Design a contact page"
**Good:** "Design a contact page for an industrial electrical company. Include form on left, map and contact details on right. Professional but approachable tone."

### ❌ Mistake 5: Too Many Styles at Once
**Bad:** "Use gradients, patterns, 3D, glassmorphism, and neumorphism"
**Good:** "Primary style: glassmorphism cards on gradient backgrounds. Use 3D sparingly for product showcases only."

## Iteration Strategy

### Round 1: Structure & Layout
- Focus on placement and proportions
- Ignore colors and details
- Get the bones right

### Round 2: Visual Style
- Apply color scheme
- Add typography styles
- Define component shapes

### Round 3: Details & Polish
- Add hover states
- Specify animations
- Include micro-interactions

### Round 4: Responsiveness
- Request mobile versions
- Tablet adaptations
- Touch-specific interactions

### Round 5: Edge Cases
- Empty states
- Loading states
- Error states
